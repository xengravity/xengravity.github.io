Amazon Pay Remote Checkout Demo

Interstitial Page

The purpose of this page is to show that the SKU ID and other parameters were passed from the platform, through the Amazon Pay login process via OAuth2 state parameter, and landed on the interstitial page with the values still accessible.

  • This page will need to be hosted by the merchant on the same domain as the checkout URL
  • The merchant will need to set the "amazon_Login_accessToken" and "amazon_Login_state_cache" cookies on this page to keep the user session alive on the merchant website ( view source for sample code )
  • This page will be virtually invisible to the customer as it will immediately redirect to the checkout URL
  • In this demo you will be redirected to the checkout URL with the SKU ID and other parameters passed in the URL. The merchant will need to take these values and create their own functionality to add the item to the cart
  • Alternatively, the merchant can take the SKU ID from this page, call your add to cart API, then redirect to the checkout URL upon success
